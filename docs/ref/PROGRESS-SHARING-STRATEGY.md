## 전 단계 공유가 실패한 이유 (진단)

1. **서버 종속**: 관제탑(8095)·프리뷰(8062)·드라이브(8063)는 살아있는 서버가 필요 — 세션이 끝나면 링크가 죽는다.
2. **산재**: 진행 증거가 experiments/<id>/reports/ 수십 폴더에 흩어져 "지금 어디까지 왔나"를 한 화면으로 못 본다.
3. **일회성 preview/index.html 남발**: 실험마다 임시 HTML을 만들어 루트/실험 폴더가 오염되고 (이번에 47장 스크린샷 아카이브), 어느 것이 현행인지 불명.

## progress.html 규약

- **단일 자기완결 파일**: CSS 인라인, 썸네일은 축소 base64 인라인 (장당 ≤50KB) — 첨부/이동해도 깨지지 않음.
- **생성기**: `scripts/build_progress_html.py` (결정론적 — LLM 판단 없음). 입력 = PROJECT-STATUS.md + 최신 pipeline_run_report.json + P5 리포트 7종 + H게이트 상태 + 대표 캡처(드라이브 스크린샷·컨택트시트).
- **내용 4절**: ① 현재 단계 (P0~H2 진행 바) ② P5 7종 PASS/FAIL 표 ③ 최신 시각 증거 (썸네일 3~6장) ④ 다음 행동·게이트 대기 항목.
- **갱신 시점**: 파이프라인 각 stage_completed에서 자동 재생성 + `python3 scripts/build_progress_html.py` 수동 호출. 항상 같은 경로 덮어쓰기 (이력은 git 커밋이 담당).
- **금지**: 실험별 일회성 progress HTML 신설 금지 — 전부 이 한 파일로. 일회성 시각 증거는 experiments/<id>/reports/에 PNG로.
